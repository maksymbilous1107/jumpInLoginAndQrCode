  <name>Configure TypeScript path aliases for @/ imports</name>
  <files>tsconfig.json</files>
  <action>
Update tsconfig.json to configure @/ path aliases matching Vite behavior (maps to project root):

1. Read current tsconfig.json (created by create-next-app in Plan 01)
2. Ensure compilerOptions includes:
   - `"baseUrl": "."` (makes paths relative to project root)
   - `"paths": { "@/*": ["./*"] }` (maps @/ to project root, NOT app/ or src/)
3. Verify other Next.js TypeScript settings are preserved:
   - `"plugins": [{ "name": "next" }]` (Next.js TypeScript plugin)
   - `"jsx": "preserve"` (App Router requirement)
   - `"moduleResolution": "bundler"` (Next.js default)

Expected tsconfig.json structure:
```json
{
  "compilerOptions": {
    "target": "ES2022",
    "lib": ["ES2022", "DOM", "DOM.Iterable"],
    "allowJs": true,
    "skipLibCheck": true,
    "strict": true,
    "forceConsistentCasingInFileNames": true,
    "noEmit": true,
    "esModuleInterop": true,
    "module": "ESNext",
    "moduleResolution": "bundler",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "jsx": "preserve",
    "incremental": true,
    "plugins": [{ "name": "next" }],
    "baseUrl": ".",
    "paths": {
      "@/*": ["./*"]
    }
  },
  "include": ["next-env.d.ts", "**/*.ts", "**/*.tsx", ".next/types/**/*.ts"],
  "exclude": ["node_modules"]
}
```

IMPORTANT: @/* maps to "./*" (project root), matching Vite config where `path.resolve(__dirname, '.')` points to root. This allows imports like `@/components/Dashboard` and `@/lib/types` to work.

Per research: Next.js automatically recognizes baseUrl and paths configuration. No additional Next.js config needed.
