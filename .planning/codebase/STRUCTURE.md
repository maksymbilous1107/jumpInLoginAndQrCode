## Directory Purposes

**Project Root:**
- Purpose: Contains all source code, configuration, and build artifacts
- Contains: TypeScript/TSX files, configuration files, compiled output
- Key files: `index.html`, `index.tsx`, `App.tsx`

**components/**
- Purpose: Houses reusable React components
- Contains: Functional React components that compose the UI
- Key files: `GlassCard.tsx` (layout component), `Dashboard.tsx` (main view), `QRScanner.tsx` (camera modal)

**node_modules/**
- Purpose: Third-party dependencies installed by npm
- Contains: React, Vite, TypeScript, Lucide icons, html5-qrcode, etc.
- Status: Git ignored

**dist/**
- Purpose: Production build output from Vite
- Contains: Bundled JavaScript, HTML, and assets
- Status: Git ignored (generated by `npm run build`)

## Key File Locations

**Entry Points:**
- `index.html`: HTML document that loads the application and defines global styling
- `index.tsx`: React bootstrapping that mounts App to DOM
- `App.tsx`: Main React component containing all state and auth logic

**Configuration:**
- `tsconfig.json`: TypeScript compiler options, path aliases (@/*), JSX settings
- `vite.config.ts`: Vite dev server and build config, path resolution, environment vars
- `package.json`: Dependencies (React 19, Lucide icons, html5-qrcode), build scripts

**Core Logic:**
- `App.tsx`: Authentication flows (login/register), state management, form handling
- `components/Dashboard.tsx`: User profile display and check-in interface
- `components/QRScanner.tsx`: QR code scanning via html5-qrcode library

**Type Definitions:**
- `types.ts`: Interfaces for UserProfile, AuthState, SchoolOption
- `constants.ts`: School list (RIMINI_SCHOOLS), color constants (JUMPIN_COLORS)

**Styling:**
- `index.html` <style> section: Global CSS for glass effects, animations, color schemes
- Tailwind CSS: Applied via CDN in index.html and used throughout JSX
- Per-component classes: Tailwind utilities in className attributes

## Naming Conventions

**Files:**
- Components: PascalCase with .tsx extension (GlassCard.tsx, Dashboard.tsx)
- Logic/data: camelCase with .ts extension (types.ts, constants.ts, vite.config.ts)
- Index files: index.html (HTML), index.tsx (React entry)

**Directories:**
- Components directory: lowercase plural (components/)
- Namespaced by feature: components/ groups all UI components
- Utilities would go in: utils/ or helpers/ (not yet created)

**Functions:**
- Event handlers: `handle` prefix (handleLogin, handleRegister, handleLogout, handleCheckIn)
- React components: PascalCase (App, Dashboard, GlassCard, QRScanner)
- Hooks: camelCase (useState, useEffect, useRef)

**Variables:**
- State variables: camelCase (authState, user, isLoading, loginError, formData)
- Constants: UPPER_SNAKE_CASE (RIMINI_SCHOOLS, JUMPIN_COLORS)
- Props interfaces: PascalCase with Props suffix (GlassCardProps, DashboardProps, QRScannerProps)

**Types:**
- Interfaces: PascalCase (UserProfile, AuthState, SchoolOption)
- Union types: camelCase or PascalCase depending on context (AuthState)

## Where to Add New Code

**New Feature (e.g., event scheduling):**
- Primary code: Create `components/EventScheduler.tsx`
- Tests: Create `components/EventScheduler.test.tsx`
- Types: Add interfaces to `types.ts` if needed
- State: Add state variables to `App.tsx` if feature is top-level
- Constants: Add to `constants.ts` if needed

**New Component/Module (e.g., user settings):**
- Implementation: `components/UserSettings.tsx`
- Props interface: Define at top of file (UserSettingsProps)
- Import in parent: Add to `App.tsx` or appropriate parent component
- Styling: Use Tailwind classes in JSX; global styles in index.html if needed

**Utilities/Helpers:**
- Create `utils/` directory if not exists
- File: `utils/[functionName].ts` (e.g., utils/validateEmail.ts)
- Export from file, import in components that need it
- Keep pure functions without React dependencies

**API/Service Layer (when backend added):**
- Create `services/` directory
- File: `services/[resource]Service.ts` (e.g., services/userService.ts)
- Fetch logic isolated from components
- Consider using axios or fetch wrapper for consistency

**Styling Extensions:**
- Global styles: Edit `<style>` block in `index.html`
- Component-scoped: Use Tailwind classes in JSX className attributes
- Complex animations: Add to index.html <style> section or separate CSS file

## Import Path Aliases

**Current alias configuration (from tsconfig.json and vite.config.ts):**
- `@/*` resolves to project root
- Example: `import App from '@/App'` imports from `/Users/fattorcomune/Programming/vibes/jumpInLogin/App.tsx`
- Used for absolute imports avoiding relative `../../../` paths

**Usage pattern in current codebase:**
- Most imports use relative paths: `import { GlassCard } from './components/GlassCard'`
- Some use @ alias: Components should prefer `@/components/ComponentName` for consistency

## Code Organization Principles

**Functional Organization:**
- Components grouped by feature area (all UI in components/)
- Shared logic stays in App.tsx (not yet refactored into separate hooks/utilities)

**Single Responsibility:**
- GlassCard: Only styling wrapper
- Dashboard: Profile display and check-in UI
- QRScanner: Camera and QR decoding
- App: State and routing

**Type Safety:**
- All props interfaces defined (GlassCardProps, DashboardProps, etc.)
- User data strictly typed (UserProfile interface)
- Auth states discriminated (type AuthState = 'login' | 'register' | 'dashboard')
